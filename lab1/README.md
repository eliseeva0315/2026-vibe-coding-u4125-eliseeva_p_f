# Telegram-бот ателье мужских костюмов

Бот помогает сотрудникам ателье ориентироваться в команде: находить коллег по отделам, просматривать контакты, отслеживать дни рождения и важные события, а также вести общий список пожеланий (wishlist).

---

## Возможности

| Команда | Описание |
|--------|----------|
| `/start` | Приветствие и клавиатура главного меню |
| `/employees` | Просмотр сотрудников через выбор отдела (кнопки), затем выбор сотрудника |
| `/important_dates` | Дни рождения коллег и важные даты в ближайшие 30 дней |
| `/wishlist` | Просмотр пожеланий и добавление нового |
| `/help` | Краткая справка |
| `/cancel` | Отмена текущего диалога |

---

## Особенности реализации

- Используется JSON (`data.json`) для хранения сотрудников, событий и wishlist
- SQLite (`activity.db`) используется для логирования действий пользователей
- Реализованы диалоги через `ConversationHandler`
- Используются кнопки (`ReplyKeyboard` и `InlineKeyboard`) вместо текстового ввода
- Добавлена нормализация отделов для исключения дублей (например, "консультант" → "консультанты")
- Данные перечитываются при каждом запросе, что упрощает обновление информации

## Требования

- Python 3.10 или новее  
- Установленные зависимости из `requirements.txt`  
- Токен бота от [BotFather](https://t.me/BotFather)

## Установка и запуск

1. Перейдите в каталог с ботом:

```bash
cd lab1
Создайте виртуальное окружение (рекомендуется) и установите зависимости:
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
Скопируйте .env.example в .env:
copy .env.example .env

Укажите в .env:

BOT_TOKEN=ваш_токен
Запустите бота:
python bot.py

После запуска в консоли появится сообщение о старте.
Откройте Telegram, найдите бота и отправьте /start.

Настройка данных

Файл data.json содержит:

employees — сотрудники (имя, роль, отдел, специализация, опыт, контакты, день рождения)
important_dates — корпоративные и важные события
wishlist — список пожеланий сотрудников

Формат даты рождения: ГГГГ-ММ-ДД.

Изменения в файле применяются без перезапуска бота (данные перечитываются при каждом запросе).

Структура проекта
lab1/
├── bot.py — основной код бота  
├── data.json — данные сотрудников, событий и wishlist  
├── activity.db — база SQLite с историей действий  
├── requirements.txt — зависимости  
├── .env.example — пример переменных окружения  
├── .env — токен бота (не добавляется в git)  
└── README.md — инструкция по запуску
Замечания по безопасности
Не публикуйте файл .env и не добавляйте токен в репозиторий
Токен даёт полный доступ к управлению ботом
При утечке токена необходимо отозвать его через BotFather
