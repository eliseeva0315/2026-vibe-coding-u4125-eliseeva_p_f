# Лабораторная работа №2  
## Подключение бота к данным

### Цель работы

Научиться подключать Telegram-бота к внешним источникам данных и использовать их в работе через программную логику.

---

### Описание решения

В рамках лабораторной работы был доработан Telegram-бот, разработанный в лабораторной работе №1.

Основное улучшение — подключение внешнего источника данных в формате CSV.

Теперь бот использует несколько источников данных:
- CSV-файл для хранения сотрудников
- JSON-файл для хранения динамических данных
- SQLite-базу данных для логирования активности

### Обоснование выбора источника данных

В качестве источника данных был выбран CSV-файл, так как он прост в использовании, не требует подключения к внешним сервисам и позволяет легко изменять данные без изменения кода.

Данный формат хорошо подходит для хранения табличной информации о сотрудниках.

---

### Используемые источники данных

#### 1. CSV-файл (employees.csv)

Основной источник данных о сотрудниках.

Структура файла:
name,department,role,email,phone,birthday

Пример записи:

Павел Орлов,консультанты,консультант,pavel@atelier.com,+7-900-111-33-11,1978-02-15


CSV используется для:
- получения списка сотрудников
- поиска по имени
- фильтрации по отделам
- получения email

---

#### 2. JSON-файл (data.json)

Используется для хранения:
- списка пожеланий (wishlist)
- важных дат и событий

Этот файл содержит изменяемые данные, которые добавляются пользователями во время работы бота.

---

#### 3. SQLite (activity.db)

Используется для логирования:
- действий пользователей
- взаимодействия с ботом

---
