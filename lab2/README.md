# Telegram-бот ателье мужских костюмов

Бот помогает сотрудникам ателье ориентироваться в команде: искать коллег, смотреть контакты, дни рождения, события и вести wishlist.

## Команды

| Команда | Описание |
|---|---|
| `/start` | Приветствие и главное меню |
| `/employees` | Показ сотрудников по отделам (кнопки) |
| `/find [имя]` | Поиск сотрудника по имени |
| `/department [отдел]` | Показ сотрудников отдела |
| `/email [имя]` | Получить email сотрудника |
| `/important_dates` | Дни рождения и важные даты на 30 дней |
| `/wishlist` | Просмотр и добавление пожеланий |
| `/help` | Справка |
| `/cancel` | Отмена текущего диалога |

## Источники данных

- `employees.csv` — основной источник сотрудников (Lab2).
- `data.json` — важные даты и wishlist.
- `activity.db` — SQLite-лог активности пользователей.

CSV-структура:

```csv
name,department,role,email,phone,birthday
```

Поиск в командах `/find`, `/department`, `/email` выполняется без учета регистра.
Если `employees.csv` отсутствует или пустой, бот сообщает понятную ошибку.

## Установка и запуск

```bash
cd lab1
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env
```

В `.env` укажите:

```env
BOT_TOKEN=your_bot_token_here
```

Запуск:

```bash
python bot.py
```

## Структура папки

- `bot.py` — логика бота
- `employees.csv` — сотрудники (CSV)
- `data.json` — события, wishlist, резерв сотрудников
- `activity.db` — SQLite-активность
- `requirements.txt` — зависимости
- `.env.example` — пример переменных окружения
- `README.md` — инструкция

## Ошибки и обработка

В коде есть базовая обработка ошибок:
- отсутствует или пустой `employees.csv`;
- некорректные аргументы команд (`/find`, `/department`, `/email`);
- сотрудник/отдел не найден;
- некорректные данные при добавлении сотрудника в wishlist-диалоге.
